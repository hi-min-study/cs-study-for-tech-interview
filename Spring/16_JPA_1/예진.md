# 면접 질문

1. JPA와 Hibernate의 차이점은 무엇인가요?
    
    **JPA는 자바 어플리케이션에서 관계형 데이터베이스를 사용하는 방식을 정의한 인터페이스**이다. 
    
    Hibernate는 **JPA라는 명세의 구현체**이다. 
    
2. JPA에서 영속성 컨텍스트란 무엇인가요?
    
    영속성 컨텍스트는 엔티티를 영구 저장하는 환경이다. 엔티티 매니저를 생성하면 자동으로 영속성 컨텍스트가 생성되고 엔티티를 관리 혹은 보관할 수 있다.
    
3. JPA에서 영속성 컨텍스트의 특징은 무엇인가요 ?
    - 1차 캐시영속성 컨택스트 내부에는 1차 캐시라고 불리는 캐시를 가지고 있습니다. **영속상태의 엔티티는 모두 1차 캐시에 저장**되고, 1차 캐시는 @Id를 키로 가지고 있는 Map이 존재한다. 엔티티를 조회할 때 바로 DB에 접근하는 것이 아니고 1차 캐시에 있는 데이터를 먼저 `조회한 후 없는 경우`에만 **DB에 접근**하여 조회 후 **다시 1차 캐시에 저장한**다.즉, 먼저 DB에 접근하는 것이 아닌 `1차 캐시에 먼저 접근`함으로서 데이터의 결과를 빠르게 가져올 수 있다.
    - 동일성 보장1번 특징과 연관되며 모든 엔티티의 데이터들은 1차 캐시에 저장되어지기 때문에식별자가 동일한 엔티티의 경우 동일성이 보장된다. 여기서 동일성이란 같은 객체를 참조한다는 의미
    - 트랜잭션을 지연하는 쓰기지연트랜잭션은 DB에서 하나의 작업 단위를 나타냅니다. 영속성 컨텍스트에서 DML이 발생했을 때 바로 DB에 저장하지 않고, **트랜잭션이 커밋될 때** 영속성 컨텍스트의 **쓰기지연 SQL 저장소**에 모아둔 쿼리들을 한 번에 저장한다. 이때 쿼리들은 영속성 컨텍스트에 따로 저장이 되며 커밋을 실행하게 되면 flush를 통해 쿼리들을 DB에 저장하게 되고 최종적으로 commit을 하여 DB에 쿼리를 반영합니다.즉, DB에 접근하는 횟수가 줄어들기 때문에 성능면에서 뛰어난다.
    - 변경 감지영속성 컨텍스트의 1차 캐시에는 스냅샷을 통해 엔티티의 변경을 감지합니다. 변경감지는 오직 영속 상태의 엔티티에만 적용이 된다.
        1. 트랜잭션을 커밋하면, flush가 호출되고, 엔티티와 스냅샷을 비교해서 변경된 엔티티를 찾는다.
        2. 변경된 엔티티가 존재하면, 쿼리를 생성해서 쓰기지연 SQL 저장소에 저장한다.
        3. 쓰기지연SQL 저장소에 생성된 쿼리들을 데이터베이스에 flush하고 commit 한다.
4. JPA에서 Entity의 Life Cycle은 어떻게 이루어지나요?
    - 비영속(new / transient) : 영속성 컨텍스트와 전혀 관계가 없는 상태
    - 영속(managed) : 영속성 컨텍스트에 저장된 상태 (엔티티 매니저를 통해 저장)
        - 영속성 컨텍스트가 관리하는 엔티티를 영속 상태라고 함
    - 준영속(datached) : 영속성 컨텍스트에 저장되었다가 분리된 상태
        - em.datach(), em.close(), em.clear() 호출 시 영속성 컨텍스트가 관리하던 영속 상태의 엔티티는 준영속 상태가 됨
        - 준영속, 분리된 상태가 되면 변경을 추적하지 않음
    - 삭제(removed) : 삭제된 상태
        - 엔티티를 영속성 컨텍스트와 데이터베이스에서 삭제
        - em.remove(entity);
5. JPA에서 Flush() 메서드는 무엇을 하는 메서드인가요? 
    
    플러시는 영속성 컨텍스트의 변경 내용을 데이터베이스에 반영한다.트랜잭션 커밋이 일어날 때 플러시가 동작하는데, 쓰기 지연 저장소에 쌓아 놨던 INSERT, UPDATE, DELETE SQL들이 데이터베이스에 날라간다.
    
6. JPA에서 Lazy Loading과 Eager Loading의 차이점은 무엇인가요?
    - 즉시로딩 : 엔티티를 조회할 때, 연관된 엔티티도 함께 조회한다. (Question을 조회할 때, List 도 조회)
    - 지연로딩 : 연관된 엔티티를 실제 사용할 때 조회한다. (Quesion을 조회할 때, List도 사용한다면 그 때만 조회)
